Odoo 17 Consignee Transfer Management (Community)

Module allows to transfer items to your consignee and when items are used by consignee than make them invoice for that items using sale order.

Note: This module is tested and working in Odoo community edition but not tested in Enterprise version. And we are supporting Ubuntu OS and not Windows OS.

Available for Enterprise

Features

  • Create Consignee as internal user.
  • Assign location to consignee record.
  • Store manager can transfer products to consignee with consignee transfer.
  • Consignee can request for return transfer to store manager with consignee return transfer.
  • One consignee can transfer products to another consignee with consignee to consignee transfer.
  • Store manager can import consignee transfer, consignee return transfer and consignee to consignee transfer using xls sheet.
  • Create Sale order for consume items by consignee.
  • Reports, for consignee location product status.
  • Product expiry notification mail to consignee.
Important Note: For xls file import we have added sample xls files into 'wizard' folder. Follow structure of file to import data direct into odoo.

How to use

  • Create user as consignee and related partner will become consignee and location will be assign automatically.


  • Create partner as consignee and location of consignee to link.


Consignee Transfer

  • Form Inventory menu, user can create consignee transfer record and make internal transfer.
  • Click on request button to make request to consignee that items are ready for transfer to his location.


  • Consignee will confirm the request for incoming transfer.


  • Once transfer is confirm. Create internal transfer using ready to transfer button.




Import Consignee Transfer



Consignee Return Transfer

  • Consignee can request to store manager to return some items.


  • Store manager will confirm return transfer request by clicking on confirm button.


  • Once store manager will confirm return transfer. Create internal picking.




Import Consignee Return Transfer



Consignee to Consignee Transfer

  • One consignee can request for another consignee for items.


  • From consignee will confirm the request.


  • From consignee will create internal picking for to consignee.


Import Consignee to Consignee Transfer



Consume Items

  • When consignee is consuming items. Make entry of it and as per need make sale order from it and make invoice consignee for consume items.
  • Make confirm and done to make sale order from consume items.






Import Consume Items



Module Configuration

  • 'Create Sale order as Confirm': from consume items, when sale order is creating if you want to make that sale order as confirm.
  • 'Create SO on Consume Item Import': When we are importing consume items from xls sheet, if we want to create direct sale order from it.
  • 'Send Mail Notification on Sale order Create': When sale order is creating from consume items. IF you want to send mail to your consignee that sale order is created for his/her consume items.


  • "Create Consignee Location" : If you want to create consignee location automatically when consignee is creating. Enable this configuration.
  • Different Mail configuration is available when making request, confirm of transfer record.
  • "Send Alert for Product Expire to Consignee": If you want to send mail notification to your consignee that product is expiring in his/her location. Enable this option.
  • From schedule actions --> 'Consignee Product Expiry' scheduler will send mail to all your consignee. Where found product is going to expire.


Reports
